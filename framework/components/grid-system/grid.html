The Grid

Create powerful multi-device layouts quickly and easily with the 12Column grid.

The Basics

The grid is built around two key elements: rows and columns. For layouts to work properly, always put your page content inside a rowMain (no, not the noodle) and a column. Our grid system doesn't rely on fixed widths. They are designed with the "fluid" mindset and are percentages of the window or device width.

Technical Details

Like all things with this framework, grids are built with box-sizing: border-box property. In my opinion, this fixes a long standing BUG in the layout system of older browsers. By using this property, we can make building grids nice and easy.

Margins are not used in our grid system. We use padding to create the gutters, and it's all based on the vertical rhythm we discussed above: 20px unit. Grids are created by a parent-child relationship between rows and columns respectively. The parent, or row, should have a class of row (easy enough). Then, you just declare how wide you want your column: fourColumns, sixColumns or how about oneColumn?

You will notice that all rows (not the rowMain) actually extend further than it's parent row. That's purposefully designed. Since columns have to have gutters, you need to remove the gutter (using negative margins) when nesting rows or the grid will be misaligned. This is normal behavior for a grid system.

Spaced Grid

Out grid system does not use float at all. Actually, this whole framework doesn't have a single float property in it! This grid uses display: inline-block and has a very small bit of space between elements. 99% of the time, this should not be an issue if you're designing sites in a modern way (stop slicing in Photoshop and re-piecing in the browser!). If, for some reason, you do need seamless grids, you can try the table-cell based grid system below, but it's not as easy to work with as this one, but still easier than float-based layouts.

Offsets

Offsets allow you to create additional space betweenColumns in a row. Like the rest of the grid, they're nestable. To do this, just add the class offsetBy_one, offsetBy_two or any other number of offsets.

Block Grids

Block grids are ul with a twoUp, threeUp, fourUp and fiveUp class. These are ideal for blocked-in content generated by an application, as they do not require rows or even numbers of elements to display correctly.
